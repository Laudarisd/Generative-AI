# Best Practices and Checklists

Good HPC habits prevent expensive mistakes.

## Before Submitting a Job

- verify environment
- verify dataset paths
- verify output directory
- set random seeds when needed
- test on a small run first

## During Training

- watch utilization
- watch loss
- confirm checkpoints are written
- monitor disk and memory

## After Failure

- inspect logs
- inspect scheduler messages
- confirm whether failure was code, memory, storage, or timeout

## Summary

Checklists reduce repeated avoidable failures.
